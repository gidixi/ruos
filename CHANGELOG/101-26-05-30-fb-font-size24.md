# 101 — framebuffer console: font Size24 + pixel store packato

**Data:** 2026-05-30

## Cosa
- `kernel/src/console/font.rs`: `FONT_HEIGHT` da `Size16` a `Size24`
  (≈1.5× più alto, ≈1.75× più largo per via del raster monospace).
- `kernel/Cargo.toml`: abilitata la feature `size_24` di
  `noto-sans-mono-bitmap` (le size disponibili sono opt-in).
- `kernel/src/console/fb.rs`: `pixel_write` ora fa un singolo
  `write_volatile::<u32>` quando `bpp == 32` invece di 3 store byte +
  un eventuale 4° (la layout LE x86 coincide con la sequenza
  byte-per-byte). Address è 4-byte aligned (pitch + `x*4`), nessun
  trap aggiuntivo su MMIO QEMU.
- `Makefile`: `run-test` timeout 120s → 240s. Il boot disegna ~2.6×
  più pixel per glifo di prima e i `.wasm` tools restano lenti
  sull'interprete `wasmi`; la finestra precedente non bastava più.

## Perché
Su VirtualBox e su monitor HiDPI il Size16 risultava troppo piccolo.
Size24 (con l'AA introdotto in [[100-26-05-30-fb-antialias]]) è
nettamente più leggibile mantenendo proporzioni corrette del raster
monospace di Noto Sans. La micro-ottimizzazione di `pixel_write`
ammortizza parte del costo extra del font più grande senza toccare
l'API.

`make run-test` torna verde con timeout esteso (TEST_PASS).
