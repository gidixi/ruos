# 138 — Task 3: Intel e1000 driver

**Data:** 2026-05-29

## Cosa

`kernel/src/net/nic/e1000.rs` — driver native per Intel 82540EM
(QEMU `-device e1000` default). Poll-mode (no IRQ), 64 slot RX + 64
slot TX su `ring::DescRing`. ~250 righe.

Init sequence (8254x SDM §14.3 / §14.5):
1. Maschera IRQ via IMC, clear ICR
2. CTRL.RST=1 → wait clear (~1-2us, bounded loop 1M iter)
3. CTRL.SLU=1 (link up; ASDE chip-side in QEMU)
4. Zero MTA (128 dword multicast hash)
5. Read MAC da RAL/RAH (no EEPROM lettura)
6. Inizializza descriptor RX/TX con phys buffer
7. Program RDBAL/RDBAH/RDLEN/RDH/RDT, TDBAL/TDBAH/TDLEN/TDH/TDT
8. TIPG default (IEEE 802.3: IPGT=10, IPGR1=8, IPGR2=6)
9. RCTL = EN|BAM|SECRC|BSIZE_2K; TCTL = EN|PSP|CT_DEF|COLD_DEF

`RxDesc`/`TxDesc` legacy 16-byte `#[repr(C)]`. Volatile read/write su
slot via `DescRing::slot(i)`.

`impl phy::Device for E1000`:
- `receive`: head slot → check status.DD, copia in Vec<u8>,
  recycle (clear status + buf_phys, bump RDT, advance head)
- `transmit`: sempre Some; TxToken::consume scrive buffer + length +
  CMD=EOP|IFCS|RS, bump TDT. Queue full → drop + bwarn (no blocking)

Wire-up `Nic` enum:
- `Nic::E1000(E1000)` variant
- `impl Device for Nic` enum-dispatch su `mac()`, `capabilities`,
  `receive`, `transmit`
- `NicRxToken`/`NicTxToken` enum dispatch su `consume`

`probe_and_init()` ora chiama `E1000::find_and_init()` se kind=E1000,
log warning per altre kind (driver non ancora wired).
