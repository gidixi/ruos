# 150 — Step 16 SSH server spec + plan

**Data:** 2026-05-30

## Cosa

Spec + plan combinato per Step 16 SSH server:
`docs/superpowers/specs/2026-05-30-rust-step16-ssh-design.md`.

Obiettivo: client OpenSSH può fare:
- **Phase 1**: `ssh root@<ruos> /bin/echo.wasm hello` → exec
  non-interactive, stdout via SSH channel data, exit code SSH exit-status
- **Phase 2**: `ssh root@<ruos>` → PTY pair + shell.wasm interactive

Auth: solo `ssh-ed25519` pubkey, `authorized_keys` su `/mnt/etc/ssh/`.
Host key: Ed25519 32-byte raw seed persistito a `/mnt/etc/ssh/host_key`.

Componenti:
- `kernel/src/ssh/{mod,hostkey,authkeys,server,channel,sunset_io}.rs`
- Crate `sunset` (no_std + alloc) + `ed25519-dalek` + `sha2` + `chacha20poly1305`
- Boot wire: `phases::userland::init` → `ssh::server::spawn()`
- Listen TCP/22 via `net::sockets::POOL`
- Single-session iniziale, accept loop con re-bind socket per ciclo

9 task TDD: skeleton + spec → host key → authkeys → sunset spike →
accept loop → pubkey auth → exec channel → PTY channel → docs.

Test gate (run-test): `ssh listening on 0.0.0.0:22`.
Test integration (run-ssh-test, nuovo): QEMU hostfwd + `ssh -p 2222
-i key root@127.0.0.1 /bin/echo.wasm hello` → exit 0 + `hello\n`.

## Crate choice

`sunset` (no_std, ChaCha20-Poly1305 + curve25519-sha256 + ssh-ed25519)
vs `russh` (tokio std-only). `sunset` vince per fit no_std + embedded
focus.

## Non-goals

No SFTP/SCP, no X11/agent forwarding, no password auth, no compression,
no IPv6, no multi-session iniziale, no rekey.
