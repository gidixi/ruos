# 159 — Pipes feature: kernel Pipe + vfs::pipe() + concurrent pipeline + shell `|` + integration test

**Data:** 2026-05-30

## Cosa

Implementazione completa delle pipeline shell nel kernel ruos (Tasks 1-7):

### Task 1 — Kernel `Pipe` anonima (`kernel/src/pipe.rs`)
Struttura `Pipe` con ring-buffer condiviso (Arc<Mutex<>>), semantica EOF su
drop del write-end, `read`/`write` bloccanti (async). Implementa il trait
`File` del VFS.

### Task 2 — `vfs::pipe()` host function
Aggiunta a `kernel/src/vfs/mod.rs`: alloca un `Pipe`, restituisce
`(read_fd, write_fd)` al modulo WASM tramite la tabella fd della Fiber.

### Task 3 — `Fiber::bind_fd`
API su `kernel/src/wasm/fiber.rs` per collegare fd arbitrari prima del launch,
usata dal coordinatore pipeline per re-direzionare stdin/stdout delle fasi.

### Task 4 — Coordinatore pipeline concorrente (`kernel/src/wasm/pipeline.rs`)
`exec_pipeline(stages: Vec<Cmd>)`: per N stage crea N-1 pipe, lancia le Fiber
in parallelo (JoinAll), chiude i write-end delle pipe nelle fasi upstream dopo
il launch (così il downstream riceve EOF). Gestisce errori di spawn per ogni
stage in modo indipendente.

### Task 5 — Host function `exec_pipeline`
Esportata al runtime WASM in `kernel/src/wasm/host_fns.rs`: deserializza
l'array di stage dal modulo WASM e chiama il coordinatore.

### Task 6 — Shell: parsing `|`
`user/shell/src/main.rs`: il loop di input riconosce `|` e spezza la riga in
stage; se ci sono due o più stage chiama `exec_pipeline` invece del singolo
`exec`.

### Task 7 — Integration test `make run-pipe-test`
`tests/pipe-test.sh`: avvia QEMU headless, connette SSH, esegue `ls / | grep bin`,
verifica che `build/pipe.log` contenga `bin` (il pipe ha filtrato).
`Makefile`: target `run-pipe-test` (prerequisito `iso ssh-key-on-disk`).

## Perché

Le pipeline sono un primitivo Unix fondamentale per comporre tool WASM in
sequenza senza file temporanei. Abilitano pattern come `ls | grep`, `cat |
sort | uniq`, ecc., rendendo la shell ruos significativamente più utile.
Il test end-to-end garantisce che tutti i layer (kernel Pipe, VFS, WASM host
fn, shell parsing) siano integrati correttamente.
