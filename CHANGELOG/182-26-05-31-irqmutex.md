# 182 — IrqMutex<T>: spin::Mutex + interrupt-flag save/restore

**Data:** 2026-05-31

## Cosa

Added `kernel/src/sync/mod.rs` with `IrqMutex<T>` and `IrqGuard<'a, T>`:

- `IrqMutex<T>::new(val)` — `const fn` constructor wrapping a `spin::Mutex<T>`.
- `IrqMutex<T>::lock()` — saves current IF flag, disables interrupts, then
  acquires the inner spinlock; returns an `IrqGuard`.
- `IrqMutex<T>::try_lock()` — non-blocking variant; if the spinlock is
  contended, IF is restored immediately and `None` is returned.
- `IrqGuard` drop order: spinlock released first (`self.guard = None`), THEN
  IF is restored — correct ordering prevents a window where interrupts are
  re-enabled while the lock is still held.
- `Send`/`Sync` impl (unsafe) delegating exclusion guarantee to `spin::Mutex`.

`mod sync;` added to `kernel/src/main.rs` (after `mod ssh;`).

The primitive is unused until Task 5 of this feature, which converts the
~42 `without_interrupts(|| mutex.lock())` call sites that share state between
task and ISR context.

## Perché

Under SMP, `without_interrupts(|| mutex.lock())` is semantically inconsistent:
the spinlock is what provides cross-core mutual exclusion; IF-masking only
prevents same-core ISR deadlock. `IrqMutex` bundles both correctly into one
primitive, making the invariant self-documenting and safe to use from SMP-aware
call sites. Task 1 of the SMP phase-0 foundations feature.
