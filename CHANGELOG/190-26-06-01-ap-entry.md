# 190 — AP entry point (cpu/ap.rs)

**Data:** 2026-06-01

## Cosa
- Created `kernel/src/cpu/ap.rs` with `ap_entry`: the Limine-delivered Application
  Processor entry point, matching the `MpGotoFunction` signature
  `unsafe extern "C" fn(&MpInfo) -> !`.
- On entry, Limine has already placed the AP in 64-bit long mode on its own stack.
  `ap_entry` reads `extra_argument()` (the dense cpu_id encoded by the BSP in Task 4
  bringup), loads the per-core GDT/TSS via `gdt::init(cpu_id)`, loads the shared
  IDT via `idt::load()`, calls `cpu::mark_online()` to increment the online counter,
  then parks in a `hlt` loop indefinitely. No STI — APs receive no interrupts in
  Fase 1.
- Registered the new submodule with `pub mod ap;` in `kernel/src/cpu/mod.rs`.

## Perché
SMP Phase 1 Task 3: each AP needs a safe, well-defined entry point that brings the
core to a consistent state (own GDT/TSS, shared IDT, online flag set) before parking.
This entry will be handed to `MpInfo::bootstrap()` in Task 4 (bringup). The function
is deliberately minimal — no work, no IRQs — keeping all cooperative executor
activity on the BSP for Fase 1.
