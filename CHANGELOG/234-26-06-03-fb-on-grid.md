# 234 — Framebuffer console rewired on Grid + Surface pipeline

**Data:** 2026-06-03

## Cosa
Task 8 dell'engine terminale. `FramebufferConsole` non disegna più pixel
direttamente: la vecchia struct con `cur_col/cur_row/fg/bg/cols/rows` e i
metodi `put_char`, `draw_glyph`, `pixel_write`, `newline`, `scroll_up`,
`parser_advance_byte` è rimpiazzata dalla nuova che tiene `Grid + Surface +
GlyphCache + render::flush`. La funzione `self_test` è riscritta per leggere
dal back-buffer. Aggiunti a `Grid` i metodi cursor (`move_up/down/left/right`,
`goto`) e `erase_to_eol`. Aggiunta asserzione T23 a `engine_test.rs` (gated
`#[cfg(feature="boot-checks")]`). `tick_cursor` e tutti i suoi atomici rimasti
invariati.

## Perché
Integrazione del pipeline Tasks 2-7: Grid (celle + dirty tracking),
GlyphCache (maschere alpha), Surface (back-buffer + blit MMIO), render::flush
(compositing). La vecchia implementazione aggirava l'intero pipeline.
