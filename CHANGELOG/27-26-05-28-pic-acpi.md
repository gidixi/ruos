# 27 — PIC disable + ACPI parsing (MADT)

**Data:** 2026-05-28

## Cosa
- Aggiunta dep `acpi = "5"` (rust-osdev). Versione risolta: **5.2.0**
  (`acpi 6.x` disponibile ma fuori dal vincolo `^5`).
- `kernel/src/pic.rs`: `disable()` maschera tutti i 16 IRQ del 8259
  (master `0x21 = 0xFF`, slave `0xA1 = 0xFF`).
- `kernel/src/acpi_init.rs`: implementa `AcpiHandler` via HHDM, parsa l'RSDP
  che Limine fornisce (nuova `RsdpRequest`), estrae da MADT
  `lapic_base`, `ioapic_base`, lista `IrqOverride`. API
  `parse() -> Result<AcpiInfo, AcpiInitError>`.
- `kmain` chiama `pic::disable()` poi `acpi_init::parse()`, logga
  `ruos: acpi ok lapic=0x... ioapic=0x... overrides=N`.

## Perché
Prerequisito per LAPIC/IOAPIC (Task 4): servono gli indirizzi base e gli IRQ
overrides per ridirigere correttamente le linee legacy.

## Adattamenti API rispetto al plan
- `acpi 5.2.0`: `unmap_physical_region<T>` è una **funzione associata**
  (`fn unmap_physical_region<T>(region: &PhysicalMapping<Self, T>)`), non un
  metodo `&self`. Implementato come no-op coerentemente al plan.
- `PhysicalMapping::new` è `unsafe fn`: la chiamata è racchiusa in un blocco
  `unsafe { ... }` dentro `map_physical_region` (che è già `unsafe fn`).
- `Apic::local_apic_address` è già `u64` (non serve cast).
- `io_apics`, `interrupt_source_overrides` sono `ManagedSlice<'_, _, A>` che
  deref a `&[T]`: `first()`/`iter()` funzionano come da plan.
- `Polarity` / `TriggerMode` vivono in `acpi::platform::interrupt::{Polarity,
  TriggerMode}`, esattamente come scritto nel plan.
- `RsdpRespData::address` è `*mut ()` e, per le base revision diverse da 3
  (qui usiamo `MAX_SUPPORTED = 6`), è un **indirizzo virtuale** HHDM-mapped.
  `AcpiTables::from_rsdp` vuole invece un indirizzo **fisico** (che poi gira
  al nostro handler che ri-aggiunge l'HHDM offset). Sottraiamo quindi
  `hhdm_offset` dal puntatore Limine prima di passarlo a `from_rsdp`. Senza
  questa correzione il primo map vedrebbe un indirizzo virtuale duplicato
  (HHDM applicato due volte) e l'RSDP non validerebbe.

## Output seriale osservato
```
ruos: hello serial
ruos: heap ok base=0xFFFF800000100000 size=4194304
ruos: alloc box=0xCAFEBABE vec=[0, 1, 2, 3, 4]
ruos: idt up
ruos: bp ok rip=0xFFFFFFFF8001B63D
ruos: acpi ok lapic=0xFEE00000 ioapic=0xFEC00000 overrides=5
TEST_PASS
```

Su QEMU `q35` (default) i valori sono i canonici x86:
LAPIC `0xFEE00000`, IOAPIC `0xFEC00000`, 5 ISO MADT (gli override standard
delle linee ISA → GSI installati dal firmware QEMU).
