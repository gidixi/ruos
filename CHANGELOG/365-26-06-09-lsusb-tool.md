# 365 — lsusb: elenco dispositivi USB enumerati

**Data:** 2026-06-09

## Cosa

Aggiunto il tool `lsusb` (gemello di `lspci`/`lscpu`): elenca ogni device USB
enumerato dallo stack xHCI, una riga per slot:

```
Bus BB Dev SS  Port P  Tier T  ID vvvv:pppp  <speed>  <kind>
```

dove `Bus` = indice controller xHCI, `Dev` = slot id per-controller, `ID` =
idVendor:idProduct, `speed` = Full/Low/High/Super, `kind` =
Hub/Keyboard/Mouse/Msc/Other.

Modifiche di supporto:

- `UsbDevice` ora memorizza `vid`/`pid`/`class` (prima il device descriptor
  veniva letto e scartato, restituendo solo la classe per il dispatch). I campi
  sono popolati in `read_device_descriptor`.
- Nuova `registry::usb_list() -> Vec<UsbInfo>`: snapshot ordinato per
  (ctrl, slot) di tutti gli slot enumerati.
- Nuova host fn `ruos_usb_list(buf, len, used) -> errno` (modulo `ruos`,
  runtime wasmi), che ricalca esattamente `ruos_pci_list` (text pre-formattato,
  ENOBUFS=8 + dimensione richiesta in `used_ptr`).
- `lsusb` registrato come boot module diskless in `limine.conf` (come `lspci`),
  nel workspace `user/Cargo.toml`, e nei `BIN_TOOLS` del Makefile.

## Perché

Mancava un omologo USB di `lspci`: per debug bare-metal (USB input, MSC live-CD,
hub multi-tier) serviva un comando shell che mostri cosa lo stack xHCI ha
davvero enumerato, con VID:PID e topologia.
