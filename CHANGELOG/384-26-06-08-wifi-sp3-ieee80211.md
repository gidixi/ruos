# 384 — WiFi RTL8188EU SP3: 802.11 protocol layer (scan logic)

**Data:** 2026-06-08

## Cosa
`kernel/src/usb/wifi/ieee80211.rs` (nuovo) — layer 802.11 puro, indipendente dal
chip:
- `build_probe_request(sa, ssid)` — frame mgmt probe-request (header MAC 24B +
  IE SSID + IE supported-rates); ssid vuoto = wildcard/broadcast.
- `parse_beacon(frame)` — parsa beacon/probe-response: walk degli information
  element → SSID (IE 0), canale (IE 3 DS-param), sicurezza (RSN IE 48 = WPA2,
  vendor MS WPA IE 221, privacy bit nelle caps = WEP, altrimenti open).
- `ScanResult { ssid, bssid, channel, security }` + enum `Security`.

## Perché
Fondamenta logiche dello scan. A differenza degli strati registro/chip (SP1-2,
blind/non verificati), questo è **standard-spec 802.11 → corretto a prescindere**
dal transport: build/parse di frame non dipende dai registri Realtek.

## Note / stato
Non ancora pilotato via radio: serve il wrapping RTL TxDesc/RxDesc + endpoint
config bulk + channel/RF init (SP3b/SP4) per trasmettere il probe e ricevere i
beacon reali. `#[allow(dead_code)]` finché SP3b non lo usa.
