# 96 — Wire keyboard ISR + console_drain + RuntimeState stdio to PTY 0 (T3)

**Data:** 2026-05-29

## Cosa

- `kernel/src/keyboard/mod.rs`: ISR riscritta. 0xE0 prefix latchato via
  `static EXTENDED: AtomicBool`. Extended scancodes tradotti in ANSI escape
  sequences (`\x1b[A/B/C/D` per frecce, `\x1b[H/F` per Home/End, `\x1b[3~`
  per Delete). Tutti i byte (regolari e ANSI) inviati a `pty::master_input_push(0, b)`.
- `kernel/src/keyboard/queue.rs`: rimosso. `pub mod queue;` rimosso da mod.rs.
- `kernel/src/executor/mod.rs`: aggiunto `console_drain_task` (loop su
  `pty::master_output_read(0)` → scrive ogni byte su `CONSOLE.lock()`).
  Spawned in `run()`.
- `kernel/src/wasm/state.rs`: `RuntimeState::new()` apre `/dev/pts/0` tre
  volte (FD 0/1/2) via `vfs::block_on(vfs::open(...))`. Rimossa variante
  `FdEntry::Stdin`.
- `kernel/src/wasm/suspend.rs`: rimossa variante `KbdReadChar`.
- `kernel/src/wasm/fiber.rs`: rimosso arm di dispatch per `KbdReadChar`.
- `kernel/src/wasm/host/fd.rs`: rimosso branch `FdEntry::Stdin` in `fd_read`.
  Rimosso riferimento a `FdEntry::Stdin` in `fd_filestat_get`.
- `kernel/src/vfs/devices.rs`: aggiornato `ConsoleFile::read` per usare
  `pty::master_output_read(0)` invece di `keyboard::queue::read_char()`.

## Perché

Task 3 di Step 12 (PTY). Collega la catena completa:
tastiera ISR → PTY 0 master input → ldisc → slave RX → shell.wasm fd_read
shell.wasm fd_write → slave TX → ldisc → master output → console_drain_task → fb.
Elimina la legacy keyboard queue, FdEntry::Stdin e SuspendReason::KbdReadChar.
